# 重調クン（重要事項説明調査スキル）

宅建業法35条書面（重要事項説明書）の作成に必要な「調査フェーズ」を自動化する一次スクリーニング担当スキル。

**位置づけ**：重説を「書く」ためのAIではなく、重説に必要な「調査をする」AIエージェント。**自動的にオンライン実査を実行し**、調査項目の漏れチェック・取得書類の要点抽出・報告書の書式化が主な仕事。役所窓口ヒアリング・現地調査・最終的な法的判断は人が担う。

---

## 起動時の必須アクション（自動実査トリガー）

ユーザーから物件住所を受け取ったら、**ユーザーが特に指示しなくても、以下の手順を自動的に実行する**。ユーザーに「実査もしますか？」と尋ねず、デフォルトで実査まで完走する。

### Step 0：冒頭確認事項

物件住所・取引種別（売買/賃貸）・用途（居住/事業用）を受け取った後、**最初に以下のみ確認**：

1. **区分所有建物か否か**（マンション・区分店舗等）
2. **PDF書類の有無**（登記簿謄本・都市計画図・建築計画概要書・ハザードマップ等を後で添付できるか）

これら以外の質問は最低限にとどめ、すぐに実査に進む。

### Step 1：自動オンライン実査の実行（最重要・MUST）

物件住所を取得した瞬間、**Claude in Chromeを自動起動**して以下を巡回する。**「実査しますか？」と聞かない**。**ユーザーの明示指示を待たない**。

#### 1-1. 重ねるハザードマップ（国土地理院・全国共通）

URL: `https://disaportal.gsi.go.jp/maps/`

実施項目：
- 物件住所を検索バーに入力 → 該当地点を表示
- レイヤー切替で以下を順次確認：
  - 洪水浸水想定区域（想定最大規模）→ 浸水深を取得
  - 高潮浸水想定区域 → 該当有無と浸水深
  - 内水浸水想定区域 → 該当有無
  - 土砂災害警戒区域 → 警戒区域・特別警戒区域の指定有無
  - 津波浸水想定 → 該当有無
  - 地形分類（自然地形・人工地形）
- 標高（DEM5A）を読み取る
- 各レイヤーの調査結果と**確認日（YYYY/MM/DD）を必ず記録**

#### 1-2. 自治体の都市計画情報GIS

検索クエリの作り方：
- 物件所在地の **都道府県名＋市区町村名** を抽出
- ウェブ検索：`「{自治体名} 都市計画 インターネット 提供」` または `「{自治体名} 都市計画情報 GIS」`

主要自治体GISのURL（参考・優先利用）：

- **建ぺい率（%）**
- **容積率（%）**
- **防火指定**（防火地域・準防火地域・指定なし）
- **高度地区**（最高限・最低限の指定）
- **地区計画**の有無と概要
- **景観地区**・特別用途地区
- **日影規制**

#### 1-3. その他のオンライン情報

- **登記情報提供サービス**：https://www1.touki.or.jp/ （ユーザーがアカウント保有時のみ案内）
- **建築計画概要書**：自治体の建築指導課窓口確認（オンライン取得不可と明記）
- **路線価・公示地価**：https://www.chika.nta.go.jp/

### Step 2：チェックリスト生成（実値反映）

Step 1 で取得した値を、チェックリストの「**状況**」「**調査結果**」「**備考**」欄に**自動的に反映**する。

例：
| カテゴリ | 調査項目 | 状況 | 備考 |
|---|---|---|---|
| 都市計画 | 用途地域 | ✅ 商業地域（東京都都市計画情報 2026/04/26確認） | 確認済 |
| ハザード | 洪水浸水想定 | ⚠️ 浸水深0.5〜3.0m（重ねるハザードマップ 2026/04/26） | 要説明 |

未取得項目（窓口確認必須等）は「**未着手・要窓口確認**」と明記し、調査先の窓口名・住所・電話番号を備考に記載する。

### Step 3：PDF解析（書類添付時）

ユーザーが PDF を添付したら、即座に書類要点を抽出して xlsx の「書類要点」シートに反映。

### Step 4：xlsx出力

調査サマリ＋チェックリスト＋書類要点＋未確認事項の4シートで出力。各シートに**実値**を必ず入れる。

## 区分所有建物の場合の追加処理

「この物件は区分所有建物（マンション・区分店舗等）ですか？」と冒頭で確認。

**区分所有建物の場合：**

宅建業法35条書面には、管理会社が発行する**「重要事項調査報告書」**の内容を記載する必要がある。この書類はウェブ上では取得できず、**管理会社への有料申請**が必要。

| 項目 | 内容 |
|------|------|
| 書類名 | 重要事項調査報告書（管理組合・管理会社発行） |
| 費用 | 管理会社により異なるが目安 **3,300円〜5,500円（税込）** |
| 納期 | 申請から **1〜2週間程度**（会社・時期により異なる） |
| 申請方法 | 管理会社へ直接依頼、またはWeb申請フォーム経由 |
| 受領形式 | 書面郵送 または PDF電子交付（会社により選択可） |

**管理会社の連絡先の調べ方：**
1. 登記簿謄本（建物）の「管理者」欄を確認
2. 物件内掲示板・管理規約に記載の管理会社名で検索
3. 区分所有者に直接確認

**主な管理会社のWeb申請窓口：**
- 大京アステージ：https://www.daikyo-astage.co.jp/
- 長谷工コミュニティ：https://www.haseko-community.co.jp/
- 日本ハウズイング：https://www.nippon-housing.co.jp/
- 東急コミュニティー：https://www.tokyu-cm.co.jp/
- 野村不動産パートナーズ：https://www.nomura-re-partners.co.jp/

**チェックリストに追加する区分所有建物固有の調査項目：**
- 専有部分・共有部分の範囲（規約共用部分の有無）
- 管理費・修繕積立金の月額と滞納有無（過去の滞納履歴含む）
- 大規模修繕計画・修繕積立金の残高と将来計画
- 管理組合の収支・財務状況（決算書）
- 管理規約・使用細則の主要制限事項（ペット・民泊・事務所利用等）
- 建物設備の維持管理状況（エレベーター・消防設備等）
- アスベスト調査・耐震診断の実施有無
- 滞納管理費等の承継義務（新所有者が引き継ぐ場合あり）

## チェックリスト生成カテゴリ

- 登記関連（所有権・抵当権・差押え・仮登記・地役権等）
- 都市計画（用途地域・建ぺい率・容積率・地区計画・景観地区等）
- 道路・接道（建築基準法上の道路種別・幅員・セットバック要否）
- ハザード（洪水・土砂・津波・液状化・火山・内水）
- ライフライン（上水道・下水道・都市ガス・電気の引込み状況）
- 法令上の制限（農地法・森林法・国土法・土地区画整理・市街地再開発等）
- 取引条件（手付・融資特約・引渡し条件・瑕疵担保等）

## PDF読込み・要点抽出

| 書類 | 抽出する主な情報 |
|------|----------------|
| 登記簿謄本（土地） | 地番・地目・地積・所有者・担保権等 |
| 登記簿謄本（建物） | 家屋番号・種類・構造・床面積・所有者・担保権等 |
| 都市計画図 | 用途地域・防火指定・容積率・建ぺい率 |
| 建築計画概要書 | 確認番号・建築年・主要用途・施工者 |
| ハザードマップ | 各ハザードの危険度区分・浸水想定深 |

## xlsx出力構成

1. **調査サマリ**：物件概要・調査日・作成者・主要調査結果のハイライト（**実値入り**）
2. **チェックリスト**：調査項目・調査先・取得書類・調査結果・備考（**実値入り**）
3. **書類要点**：各書類の抽出情報を整理した一覧
4. **未確認事項**：要追加調査の項目リスト（優先度・対応方針・担当・期日）

## 使い方・フロー

### ケース1：チェックリスト＋自動実査
```
「東京都港区六本木3-16-12、一棟ビル売買、事業用」
→ Claude in Chrome 自動起動
→ 重ねるハザードマップで実値取得
→ 東京都都市計画情報で実値取得
→ 結果反映済みチェックリスト → xlsx出力
```

### ケース2：PDF添付込み
```
「港区〇〇の物件、PDFも添付」（PDFを添付）
→ Step 1（自動実査）＋ Step 3（PDF解析）
→ チェックリスト＋書類要点が両方実値で埋まる → xlsx出力
```

### ケース3：チェックリストのみ（高速モード）
```
「実査スキップして項目だけリストアップして」
→ Step 1 をスキップしてチェックリストの枠のみ生成
```

## 注意事項・限界

- 本スキルは「一次スクリーニング」であり、法的判断の最終確認は宅建士・不動産鑑定士が行う
- **オンライン実査は Claude in Chrome 経由で自動実行する**（v2より）
- 役所窓口でのヒアリング・現地調査・越境・境界確認は人が担う
- ハザード情報は公開データを参照するが、最新版は各自治体で要確認
- 自治体GISのURLが変更された場合、ウェブ検索で再特定する

---

## 出力スタイル

- チェックリストは「調査先・取得書類・確認ポイント・状況・備考」を明示
- **「状況」欄には ✅（確認済）・⚠️（要説明）・❌（未着手）等の記号で視認性を上げる**
- PDF要点は「重説記載に使える表現」に寄せてまとめる
- xlsxはxlskillを使って作成する（xlsxスキルが利用可能な場合はそれを参照）
- 不明点・確認要事項は色付きセルで強調
- **取得日（YYYY/MM/DD）と出典URLを必ず併記**
